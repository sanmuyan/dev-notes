# 命名规范

## 文件

文件、文件夹命名应该符合人类直觉，`user-data` `data` `test` `test-data`

## Json

```json
{
    // 字段一般使用小写字母，多个单词用下划线连接
    "server_name": "server-1"
}
```

## 数据库

库名 表名 字段名 key 名一律使用小写字母，多个单词使用下划线连接

## 功能命名

### 函数

- get_user 获取一个用户
- list_user 获取多个用户
- delete_user 删除一个用户
- create_user 创建一个用户
- update_user 更新一个用户
- decode 解码
- encode 编码
- generate_data 生成数据
- extract_data 提取数据

### 文件夹

- util 辅助函数
- service 业务逻辑
- controller 业务控制
- middleware 中间件
- model 数据模型

## 行文

- 段落以中文为主的时候，全文使用中文标点符号
- 段落以英语为主的时候，全文使用英语标点符号
- 中英混合时，汉字和单词之间用空格隔开
- 英语注释，段落首字母建议大写
- 标准输出，段落首字母建议大写
- 错误输出，段落首字母不建议大写
- 日志输出，段落首字母不建议大写
